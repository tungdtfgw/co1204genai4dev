Designing Effective System Instructions

  • Be clear and concise: state exactly what you expect the model to do.
  • Define a persona: specify tone, style or role (e.g. "You are a helpful assistant").
  • Avoid ambiguity: conflicting instructions may degrade performance.
  • Combine system instructions with user prompts for fine-grained control.

Well-crafted system instructions improve response consistency and reliability.
