Key Concepts of the Framework

  • The Agent
    An Agent is the fundamental building block, representing an AI with a specific purpose. It is defined by a name and a set of instructions (its system prompt).
  • The Runner
    The Runner is responsible for executing an agent with a given input. It manages the interaction with the OpenAI API and returns the agent's final output.
  • Handoffs
    Handoffs allow agents to pass tasks to other agents, enabling complex workflows where different specialists work together.
  • Guardrails
    Guardrails are safety mechanisms that filter and validate inputs before they are processed by agents, ensuring appropriate behavior.
  • Asynchronous Design
    The framework is built with asyncio, allowing agents to work concurrently and handle multiple tasks efficiently.
