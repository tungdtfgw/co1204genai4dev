Anthropic Claude Prompt Caching

  • Anthropic Claude supports both input and output caching.
    • Input caching is automatic and requires no configuration.
    • Output caching allows developers to control cache behavior and TTL.
  • Input caching works similarly to OpenAI's approach.
    • It automatically detects and caches unchanged parts at the beginning of prompts.
    • Requires a minimum of 1024 tokens to activate caching, with cache TTL of about 5 minutes.
  • Output caching provides more control over cache behavior.
    • Developers can specify cache TTL and cache keys.
    • This is useful for applications that need to manage cache invalidation or have specific caching requirements.
