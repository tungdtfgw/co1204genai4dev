How Caching Works

  • Prompt caching works by storing and reusing the processing results of unchanged prompt parts.
    • The prompt is analyzed to identify the fixed part (system prompt, context) and the changing part (user query).
    • After processing the fixed part, the model's internal state is stored in cache with a unique key.
    • When a new prompt with the same fixed part is received, the system loads the stored state and only processes the changing part.
  • The effectiveness of prompt caching depends on the length of the fixed part.
    • The longer it is, the more time and cost savings can be achieved.
    • The minimum threshold is usually 1024 tokens to achieve significant benefits.
