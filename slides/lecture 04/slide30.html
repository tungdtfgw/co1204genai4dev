Comparison Input vs Output Caching

  • Input Caching stores the model's internal state after processing the beginning of the prompt.
    • It helps reduce latency by 50-85% and saves API costs by 50-90%.
    • Response quality is always up-to-date, but it requires the prompt prefix to be exactly the same.
  • Output Caching stores entire prompt-response pairs for direct retrieval.
    • It helps reduce latency to nearly zero and saves costs by nearly 100% when there's a cache hit.
    • This method supports flexible matching (exact and semantic) but responses may not be updated with the latest model version.
