Input Caching Overview

  • Input caching is a method of storing the model's internal state after processing the beginning of the prompt.
    • It allows reusing this state for prompts with the same beginning.
    • This technique helps reduce processing time by 50-85% and API costs by 50-90%.
  • Technically, input caching works by storing hidden states and attention weights.
    • The beginning of the prompt must be exactly the same to achieve a cache hit.
    • Cache time-to-live (TTL) is typically from 5 minutes to 1 hour, depending on the provider.
