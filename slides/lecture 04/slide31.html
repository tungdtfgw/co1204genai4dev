OpenAI Prompt Caching

  • OpenAI implements automatic input caching for GPT-3.5 and GPT-4 models without requiring configuration.
    • The system automatically detects and caches unchanged parts at the beginning of prompts.
    • Requires a minimum of 1024 tokens to activate caching, with cache time-to-live (TTL) of about 5 minutes.
  • Technically, OpenAI uses a hash mechanism to identify identical parts between prompts.
    • When an identical prompt beginning is detected, the system loads the stored state and continues processing from there.
    • This mechanism works best with applications that have long and fixed system prompts.
